/ HISTORY
      snapshots and audit trail

    
Revisions and audit trail

    
Every promotion snapshots the revision it displaced, so a change is reversible.
      Restoring one is atomic and is itself recorded. The trail holds metadata only (action, skill,
      revision, actor, timestamp, and an optional rejection reason), never skill text.

Skills under control

    
Every skill the router serves, with the revision it is serving. SkillOpt trains
      bounded instruction edits, measures them on train and held-out tasks, and leaves each result
      quarantined for review. Skills without an eval task set can't be measured yet: a CLI run
      auto-drafts one.
