    / 05 · READ IT HONESTLY METHOD

    What the marks mean.

    Every mark on this page is defined. The rules that turn raw commands into tokens, pass/fail labels, and forks are below.

    How to read this honestly · method

    How trie tokens are made · scripts/trie_forks.py

    • Pure inspection commands (ls, cat, grep, …) are dropped: they don't change state, so they never appear here.
    • Consecutive duplicate tokens are collapsed; digits in filenames become N.
    • Compound commands key on the first word: git add -A && git commit counts only as git-add. Check the raw command when it matters.

    Where pass / fail comes from

    • A run is PASS when its test_output.txt pytest summary shows N passed and no N failed. Runs without test output show as unlabeled.

    What counts as a fork

    • A node with ≥ 2 children where one child's subtree is purely PASS and another purely FAIL. With few runs, a fork is a hypothesis, not proof.

    What this page will not do

    • Read-only. It never launches runs, walks graphs, or edits artifacts — it renders what carn already wrote to disk.
